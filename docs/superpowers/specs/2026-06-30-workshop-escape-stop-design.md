# Workshop ESC 停止設計

## 目的

現在表示中の Workshop 対話が生成中の場合、Escape キーで生成を停止できるようにする。

## 挙動

- 現在表示中の対話が生成中の場合だけ、ウィンドウの Escape キーを受け付ける。
- 入力欄にフォーカスがあっても停止する。
- 別の対話へ移動した後の Escape キーでは、バックグラウンド生成を停止しない。
- 思考、本文、ツール呼び出しのいずれかが始まっていれば、既存の停止処理で途中結果を保存する。
- AI の出力が一切始まっていなければ、送信したプロンプトを入力欄へ戻し、直前のユーザーメッセージを表示中の履歴と保存済み対話から削除する。

## 実装方針

- 既存の `stopActive` と Workshop 実行ストアを再利用し、新しい取消機構は作らない。
- 停止時に「出力前か」と、復元するプロンプトおよび送信前履歴を返せる最小の結果をストアから公開する。
- `WorkshopView` は現在の対話が生成中の間だけ `window` の `keydown` を購読し、Escape 時に停止結果を UI へ反映する。
- 出力前停止では送信前履歴を既存の会話保存 API で上書きする。

## 検証

- 出力前停止がプロンプトと送信前履歴を返すことをテストする。
- 出力開始後の停止では復元情報を返さないことをテストする。
- 現在の対話だけが Escape の対象になることをテストする。
- ルートの frontend lint と関連テストを実行する。

## 対象外

- バックグラウンド生成を別の対話から停止する操作は追加しない。
- 停止確認ダイアログや新しいショートカット設定は追加しない。
